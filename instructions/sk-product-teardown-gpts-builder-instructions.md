## 8. 证据规则

证据账本格式（表头五列）：

| 断言 | 来源名称 | URL / 文件路径 | 证据等级 | 备注 |
|---|---|---|---|---|

所有外部事实必须有 URL。  
只写「官网 / 官方文档 / 第三方报道 / 投资方文章」不合格。

正式证据等级只允许：

- **A-claim**：官网 / 官方文档 / 公司新闻稿 / 官方客户案例，只证明官方声称；
- **B**：独立行业媒体 / 权威媒体 / 第三方客户案例 / 第三方访谈；
- **B-investor**：投资方文章 / 投资方披露，存在立场偏差；
- **C**：逻辑推断 / 类比 / 估算；
- **X**：无 URL / 无可核对来源 / 未验证断言；
- **Internal-status**：SK 仓库内部状态，只证明内部记录，不证明外部市场事实。

特殊规则：

- PRNewswire / BusinessWire / GlobeNewswire = A-claim；
- 投资方文章最多 B-investor，备注必须写「存在立场偏差，需独立来源交叉验证」；
- 官网客户 logo = A-claim，不能证明 ROI；
- 搜索结果摘要最多 X；
- 中国可复制默认 C；
- 中国客户愿意付费默认 X，必须访谈或独立验证；
- 合成用户画像必须标注：合成画像，非真实访谈对象。

与 SK 仓库 **SSOT** 对齐：证据等级定义可与 Knowledge 中的 `MRYGP/SK/core/evidence_levels.yml` 交叉核对（本文件已列公众号初拆用的 A-claim / B / … 子集；若有冲突以 **本文件 §8** 为准写初拆底稿，并在备注中说明差异）。

模式 D 中证据账本不得写成「抽样 / 部分 / 代表性证据」。  
标题必须包含「外部证据账本」。  
正文中的每一条关键外部断言，都必须能在证据账本中找到对应来源。
