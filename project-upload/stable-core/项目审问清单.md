## 使用说明

1. **任何项目进入视野时**（拆解别人、评估自己、朋友带想法来聊），都按本清单逐条过

3. **顺序不能颠倒**：先反向（排除失败），再正向（寻找可能），最后身份（照镜子）
4. **任何一条 Part A 触发 → 立即否决，不进入 Part B 之后**

## Part A · 10秒排除线（触碰任何一条 → 直接否决）

> 如果这个项目触发了以下任意一条，不要继续往下看。直接否决，或者让它降级到"观察池"。

| 编号 | 排除项 | 触发方式 | 状态 |
|------|--------|---------|------|
| **A01** | 创始人把"能活"和"百倍"混淆 | 如果对话中反复出现"颠覆""百倍""绝佳机会"，且没有"月入X万养活团队"的菜地规划 | v1.0 |
| **A02** | 项目是树，但没有菜地输血 | 产品需要3年以上才挂果，创始人没有同步种菜的现金流来源 | v1.0 |
| **A03** | 创始人能力画像和项目严重错配 | FM011触发——做C端但没做过C端、需要全球化销售但没做过、需要顶级BD但是技术背景 | v1.0 |

## Part B · 反向审问（用 failure_modes 逐条过）

> **如果同时命中3个以上🔴，且包含FM003或FM015 → 直接否决**

|--------|------|---------|---------|
| **FM003** | 伪需求 | 用户用频率够吗？付费意愿验证过吗？**是"想要"还是"愿意付钱"？** | ☐ |
| **FM015** | 无飞轮 | 机制转一次，下一次成本会不会更低？**还是每次都要重新花钱？** | ☐ |
| **FM008** | 快速复制 | 护城河是什么？ChatGPT/巨头3-6个月能不能复制？**不能把"未来的网络效应"当护城河** | ☐ |

## Part C · 正向审问

### C1 · 好奇心 vs 飞轮三问（源自010拆解）

> **用户第一次来之后，还会不会再回来？**

- [ ] **C1-Q1**：好奇用完之后，用户还有没有理由回来？
- [ ] **C1-Q2**：假设每个用户只来一次，这笔生意赚不赚钱？
- [ ] **C1-Q3**：那个会留下来的用户，你见过他本人吗？他叫什么？他上周做了什么？

**检验句**：这三个问题任何一个答不出来或含糊 → 这个项目停在"好奇心层"，没到"飞轮层"。

### C2 · 种树种菜双轨自检（源自D018补丁）

- [ ] **C2-Q1**：这件事本身是菜还是树？
  - 菜的特征：3-6个月能见到第一笔真实现金流，投入产出线性可见
  - 树的特征：需要2-5年积累才能产生复利，护城河随时间增强
- [ ] **C2-Q2**：如果是树，有没有配套的菜地输血？
- [ ] **C2-Q3**：如果是菜，它的产出能为未来某棵树输血吗？
- [ ] **C2-Q4**：有没有陷入两种错配？
  - 错配A：把菜当树浇（一块菜地刚有苗头就all in讲平台故事）
  - 错配B：把树当菜吃（有潜力的树为了短期现金流被砍枝变现）

### C3 · 诊断空白四条件（源自011 MYHAIR AI拆解）

- [ ] **C3-Q1**：高焦虑的人是不是很多？
- [ ] **C3-Q2**：真正在行动的人是不是很少？（否 → 这是红海，不是诊断空白）
- [ ] **C3-Q3**：中间是不是隔着一个"没人回答的问题"？（否 → 已经有人在解这道题）
- [ ] **C3-Q4**：B端有没有人愿意为"已经准备好的客户"付钱？（否 → 没有下游买单方）

**四条件齐备 → 是真正的诊断空白**
**缺一条 → 不是诊断空白，是红海或伪机会**

### C4 · MTP五标准（如果这是一个"想做的"项目）

- [ ] **C4-S1**：能不能让读者5分钟看懂？
- [ ] **C4-S2**：有没有足够信息让读者能判断真假？
- [ ] **C4-S3**：有没有自我攻击点？（允许读者反驳）
- [ ] **C4-S4**：能不能对接具体资源人？
- [ ] **C4-S5**：能不能3个月见到现金流信号？
- [ ] **C4-S6**：你说的这套打法，是阶段一/二/三哪一种？资源储备和阶段匹配吗？（源自 `theory/DOC-D027-阶段错配.md`）

### C5 · 次生问题审问（源自DOC-D026《解决方案的次生问题定律》）

> **任何产品解决一个问题的同时，必然制造一个新问题。有接盘侠就是飞轮，没接盘侠就是炸弹。**

**先画"矛盾转移闭环图"**：
[用户痛点] → [产品方案] → [次生问题] → [接盘侠?] → [闭环?]

**然后回答"次生问题五问"**：

- [ ] **C5-Q1**：这个产品解决了A，制造的B具体是什么？谁为B买单？
  - 🔴 答案是"用户自己扛"/"我们自己扛"/"社会环境承担"
- [ ] **C5-Q2**：规模放大10倍后，B是线性增长还是指数增长？
  - 🔴 指数增长
- [ ] **C5-Q3**：有没有"自然接盘侠"主动承接B并从中获利？写出具体身份+主动动机
  - 🔴 写不出具体身份或具体动机
- [ ] **C5-Q4**：用户会不会因为B的体验而流失？
  - 🔴 会（护城河是负的）
- [ ] **C5-Q5**：政策/技术突变后，B会不会变致命？
  - 🔴 会（结构性脆弱）

**判定规则**：
- 五问全绿 → 有自闭环能力，可以开工
- 任何一问红灯 → 重新设计或找新接盘侠，不开工

**参考对照**：
- 怪兽充电宝 = 断裂案例（归还难无人接盘 → 死亡）
- 美团充电宝构想 = 闭环案例（骑手顺路主动抢单 → 飞轮）

完整定律详见：`D:\sk\theory\DOC-D026-解决方案的次生问题定律.md`

### C6 · AVP反脆弱性检验（源自CEET提炼-三条可用斧头）

> **你设计的AI产品，是让用户用得越多越依赖（认知外骨骼），还是用得越多越独立（认知内共生）？**

**核心检验句**：
> **"假设用户用这个产品6个月后停用3个月——他的独立能力会下降，还是上升？"**

- [ ] **C6-Q1**：用户离开你的产品后，能力下降还是上升？
  - 🔴 下降 = 认知外骨骼 = 长期毒性 = 监管/口碑反噬风险
  - 🟢 上升 = 认知内共生 = 长期赋能
- [ ] **C6-Q2**：用户独立后反而回来吗？回来是因为**依赖退化**还是**新能力创造了新需求**？
  - 🔴 依赖退化 → FM003 伪需求高风险
  - 🟢 新需求 → 真飞轮的底层动力
- [ ] **C6-Q3**：和C1、C5交叉验证：
  - C1好奇心型产品 ≈ C6认知外骨骼型
  - C5次生问题无接盘 ≈ C6能力退化无接盘侠
  - **三者同时🔴 → 这是一个必死的产品**

**判定规则**：
- C6全绿 + C1/C5全绿 → 长期赋能型产品，值得做
- C6🔴 → 无论C1/C5如何，都需要重新设计

完整说明详见：`D:\sk\theory\CEET提炼-三条可用斧头.md`

### C8 · 跑通者考古(先抄通,再推演)

> 这个方向有没有 1-3 个**已经跑通的真人案例**?如果有,你抄过吗?

判定:
- 🔴 没有任何跑通真人案例 → 危险姿势(可能是真空白,但更可能是方向不成立)
- 🔴 有 ≥3 个跑通案例 + 你完全没接触 → 用聪明回避抄通的辛苦
- 🟡 有跑通案例 + 你大概知道但没动手抄 → 紧迫程度未到
- 🟢 抄过至少 1 个 + 跑出 1/10 规模真实结果 → 健康姿势,可以推演差异化

来源:2026-04-25 杨提"先抄通,再推演"方法论修订(清单角度 39)

## Part D · 身份审问（照镜子）

> 这一部分最难。也最重要。

- [ ] **D-Q1**：**"如果这是一个陌生创业者发给我的BP，我会怎么拆它？"**
  - 换个名字（比如叫"某位博士的AI项目"），自己拆一遍
  - 如果换名字之后立刻发现致命问题 → 你在用自我认同回避判断
  
- [ ] **D-Q2**：**"这个项目和那位亏200万的博士，最像的5个地方是什么？"**
  - 必须写出来，写不出5个说明对照不够诚实
  - 相似点 ≥ 3个 → 高风险
  
- [ ] **D-Q3**：**"我在用'聪明'回避哪三个问题？"**
  - 010里提炼：死的创业者都是在用聪明回避关键问题
  - 自己审问：我这次有没有用复杂叠加、漂亮叙事、诱人愿景在**绕过**某个根本问题？
  
- [ ] **D-Q4**：**"我现在的姿势，是'知道自己可能错的人在公开邀请反驳'，还是'一个非常聪明的人在讲一个他相信自己一定能讲成的故事'？"**（源自KAIROS拆解）
  - 后者 = 危险信号

### D5 · 阶段错配诊断（源自 DOC-D027）

> **三问任何一问含糊 → 阶段错配高风险**

- [ ] **D5-Q1**：你这套打法属于阶段一（MVP 验证期·集中押注）/ 阶段二（商业化放大期·方法论复制）/ 阶段三（永久资本期·防守优先）哪一种？
- [ ] **D5-Q2**：你的资源储备（现金流/团队规模/方法论验证次数）匹配这个阶段吗？
  - 🔴 阶段一打法但没现金流储备 → 一次踏空就死
  - 🔴 阶段三打法但还在阶段一 → 慢性失血出局
  - 🔴 阶段二打法但方法论只验证过 1 次 → 把幸运当能力放大
- [ ] **D5-Q3**：如果你是在切换阶段，三个信号是否同时出现？（现金流信号 + 方法论信号 + 结构信号）
  - 🔴 信号没全到，但创始人焦虑或外部压力在推 → 假切换

## 六、清单的扩展机制

### 6.1 三层结构

```
草稿区（零摩擦捕获）
  ↓ 出现3次
候选区（结构化整理）
  ↓ 经过实战验证
正式清单（本文档）
```

### 6.2 三种触发器

**触发器1：每次审问完一个项目**
- 问自己："这次用到了清单里没有的判断吗？"
- 如果有，立刻写到草稿区

**触发器2：每次深度对话结束**
- 问自己（或问AI）："这次对话冒出了哪些我以前没想过的角度？"
- 立刻写到草稿区

**触发器3：每月最后一天回看**
- 草稿区 → 哪些出现3次了？升级到候选区
- 候选区 → 哪些被实战验证了？升级到正式清单
- 正式清单 → 哪些3年没触发过？下沉到历史教训库

### 6.3 三次验证规则

一条教训**必须满足以下三次触发**，才能从草稿区升级到候选区：

1. **发现**：第一次在对话/思考中冒出
2. **被引用**：第二次在另一个不同场景被用上（说明不是孤例）
3. **救了一次决策**：第三次在具体项目审问中起了作用（说明有实战价值）

只有满足三次的教训，才值得进入候选区的结构化整理。

**再经过实战验证**（有项目套用它并得到好结果），才能升级到正式清单。

### 6.4 清单条目的标准字段

进入正式清单的每一条，都要有：
- **编号**（A01-A0X / FM0XX / C-Q0X / D-Q0X）
- **一句话表述**
- **检验问题**（能套在任何项目上的那个具体问题）
- **对应FM或理论**
- **来源**（哪次对话/哪个案例提炼的）
- **状态**（v1.0自带 / vX.X新增）

### 6.5 版本变更原则

- 每次升级一条，版本号+0.1（v1.0 → v1.1 → v1.2...）
- 大改结构时，版本号+1.0
- 每次变更必须在Part G留痕
