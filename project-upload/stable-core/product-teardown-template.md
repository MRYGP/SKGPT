> 360只有漏斗（买量→变现→流失→再买量），拼多多有飞轮（更多用户→更多农户→更低价格→更多用户）。

**画出增长循环图**：
```

```
（如果画不出闭环，只能画出单向箭头，它就没有飞轮。如实写"本产品无飞轮，增长依赖___"。）

**飞轮四条件检验**：

| 条件 | 问题 | 本产品情况 |
|------|------|-----------|
| ①多边参与 | 除了平台和用户，还有哪些参与方？它们之间是否相互依赖？ | |
| ②核心价值随规模强化 | 规模扩大时核心卖点是变强还是稀释？还是越差（如广告骚扰）？ | |
| ③退出成本递增 | 用户/参与方用得越久，离开的代价是否越大？具体是什么代价？ | |
| ④单圈正收益 | 飞轮转一圈，经济模型是正的还是负的？LTV/CAC趋势如何？（如果转一圈亏钱，飞轮越快死越快——Ofo/WeWork教训） | |

**飞轮判断**：
- [ ] 真飞轮（四条件全满足）
- [ ] 半飞轮（满足1-3条，有改进空间）
- [ ] 绞肉机（①②③满足但④不满足——转得越快亏得越多）
- [ ] 无飞轮/纯漏斗（需持续买量或补贴维持增长）

**如果无飞轮**：这是一个关键风险信号。在第7章失败机制中标注"FM015-无飞轮增长依赖"。

### 9b) RRF分析链（v4.0新增，参考DOC-D022 v2.1）

> RRF = Redefine（重新定义问题）+ Rule（规则制定）+ Flywheel（飞轮构建）
> 这三步决定了一个产品的天花板：只做了R的是一次性爆款，做到RR的能站稳，做到RRF的能持续增长。
>
> ❗ 边界提醒：RRF适用于平台型/网络型商业模式。纯硬科技、专业服务、强监管行业、Pre-PMF阶段、消费硬件不适用。如本产品不在适用范围，写"本RRF不适用，原因：___"并跳过本节。

**R — 重新定义问题**

这个产品重新定义了什么问题？

| 维度 | 行业原来的定义 | 这个产品的新定义 |
|------|--------------|----------------|
| 用户 | 原来服务___ | 现在服务___ |
| 价值 | 原来解决___ | 现在解决___ |
| 边界 | 原来是___ | 现在变成___ |
| 时间 | 原来是___ | 现在变成___ |
| 成本 | 原来___付费 | 现在___付费 |
| 竞争 | 原来的对手是___ | 现在的对手是___ |

重新定义了几个维度？___个。
- 0-1个：优化，不是颠覆
- 2-3个：有颠覆潜力
- 4个以上：范式转移级别

**检验句**：这个产品的成功，是否让原来的行业领导者"看起来像在做蠢事"？
→ 是 / 否。如果"否"，它可能只是在优化，不是在重新定义。

**R — 规则制定**

它是规则套利者还是规则制定者？

- **套利信号**：封闭控制、依赖单一变现渠道、没有培育第三方生态
- **制定信号**：开放能力/API/标准、有第三方深度依赖它生存、生态在自发扩张

判断：套利者 / 制定者 / 过渡中

如果是制定者，在哪个层次？
- [ ] L1 流量规则（控制入口/分配注意力）
- [ ] L2 商业基础设施规则（供应链/交易/分配标准）
- [ ] L3 技术标准规则（协议/架构/接口）

**"消失测试"**：如果这个产品明天消失，谁会痛苦？
→ 没人痛苦 = 套利者。有人的生意依赖它 = 制定者。

**AI产品专用——依赖度检验（v2.0新增）：**
如果底层模型提供商（OpenAI/Anthropic/Google）明天把你的核心功能内置化，你还剩什么？
→ 什么都不剩 = 二阶套利者（如Jasper AI）
→ 剩工作流锁定/行业数据/用户关系 = 有防御性（如Harvey/Cursor）

**RRF综合判断**：

| 维度 | 状态 | 意味着什么 |
|------|------|-----------|
| R-重新定义 | ✅/❌ | |
| R-规则制定 | 套利/制定/过渡 | |
| F-飞轮 | 真/半/无 | |

**产品段位判断**：
- RRF全满（含第④条） = 有成为伟大公司的基因（拼多多、亚马逊）
- RRF全满但第④条不满 = 绞肉机，越快越危险（Ofo、WeWork）
- 有R无RF = 一次性爆款风险（360、Jasper AI）
- 有RF无第一个R = 做得好但不是颠覆者（守成型公司）
- 全无 = 不值得拆解（或不在RRF适用范围）

## 第 1 步：D026 五问（次生问题闭环检验）

> 详见：`D:\sk\theory\DOC-D026-解决方案的次生问题定律.md`

任何产品在解决问题 A 的同时，会制造次生问题 B。**顶级解法不是「消灭 B」，而是给 B 找到自然接盘侠（主动且有收益）**。

立项必答：

| 编号 | 问题 |
|------|------|
| Q1 | 谁为这个产品的「排泄物」（次生问题）买单？ |
| Q2 | 规模放大 10 倍后，排泄物是指数增长还是线性增长？ |

## 第 3 步：D027 三问（接盘侠性质判断）

> 详见：`D:\sk\theory\DOC-D028-接盘侠的两种性质-工业化vs手工化.md`  
> **编号说明**：三湘问道对话里本步常称「D027」；SK 仓库中 `DOC-D027` 已用于《阶段错配》，接盘侠「工业化 vs 手工化」单独立卷为 **DOC-D028**。

接盘侠存在，接下来问：**它是哪种性质？**——这决定了产品的成长曲线、护城河形态、团队规模、和能不能被 2–5 人小团队跑通。

| 编号 | 问题 | 工业化路径 | 手工化路径 |
|------|------|------------|------------|
| Q1 | 一年内你需要多少接盘侠才能验证模式？ | 数千到数万 | 数十到数百 |
| Q2 | 接盘侠之间需不需要彼此知道存在？ | 不需要 | 需要（口碑/社群） |
| Q3 | 创始人个人能不能从产品中「消失」？ | 必须能消失 | 不能消失（创始人即产品） |

**关键结论**：90% 的小团队应该选手工化接盘路径。误区是「想做亚马逊式产品但只投入了胖东来式资源」——结果什么都不是。

## 第 4 步：双榜判断（场景排序）

> 来源：2026-05-02 Yoodli 再讨论沉淀

不是单一排榜。**两张榜并行，职责不同**：

### 榜 A：产品机会榜

回答：**哪个场景最像原版商业模式？**（用于长线战略判断、未来 18–24 个月候选池）

### 榜 B：当前行动榜

回答：**哪个场景最适合我们现在写文章、找人、低成本验证？**（用于本季度/本年度动作）

**铁律**：

- 「最像原版」≠「最适合我们现在做」
- 不允许混淆。混淆会导致「以为找到了机会，实际选错了战场」
- 一个标准的输出形式：**「X 场景在产品机会榜上排第 1，但在当前行动榜上排第 3——我们当前阶段切的是 Y 场景，X 留作 18 个月后的备选」**

### 双榜实操示范（以 Yoodli 中国版为例）

| 排名 | 产品机会榜（最像 Yoodli 原版） | 当前行动榜（最适合 SK 现在做） |
|------|--------------------------------|----------------------------------|
| A1 | 金融/保险合规销售训练 | **考公/事业编面试** |
| A2 | 客服/投诉处理训练 | 留学英文面试 |
| B | 留学英文面试 | 金融/保险合规销售训练 |
| B | 企业销售培训 | 医患沟通 |
| C | 考公/事业编面试 | 律所新人答辩 |

**结论**：产品机会榜上的 A1/A2 是 Yoodli 真正的母题（企业级 AI roleplay），但 SK 现在最该做的是「考公」——这是当前行动榜的第一，适合写 MTP 招募文、找培训资源人、低成本验证。

> 🚨 **第零步：基础设施前提检验（出海/跨境项目专用，必填）**
>
> **触发条件**：凡项目涉及以下任一情况，本节必填；纯国内AI产品拆解可跳过：
> - 出海、跨境电商、境外平台（Etsy/Amazon/TikTok Shop等）
> - 国际物流、进出口业务
> - 海外用户、海外市场、境外支付和收款
>
> **为什么必填**：最重要的决策变量往往是「已经发生的政策变化」而非市场预测。实战教训：2025年5月美国取消中国商品De minimis（$800）小包免税，8个AI研究工具只有1个发现了这个已发生的变化，该变化直接把预估毛利率从65-80%拈至31-37%，整个直发模式不再成立。
>
> **强制核查四项（进入下面扫描之前必须先回答）**：
>
> | 核查项 | 问题 | 结果 | 数据来源 |
> |--------|------|------|----------|
> | 📦 小包免税门槛 | 目标市场当前对中国商品的小包免税门槛是多少？（美国/加拿大/澳大利亚/英国） | | 必须查目标市场官方海关网站，不得引用12个月前数据 |
> | 🏬 平台准入规则 | 目标平台（Etsy/Amazon/TikTok Shop）对中国大陆卖家的当前准入状态是什么？有没有新开店限制？ | | 必须查平台卖家中心官方最新政策，不得以历史经验代替当前文本 |
> | 💱 外汇和收款 | 中国团队收取境外收入的最小阻力路径是什么？（Payoneer/香港公司/其他） | | 检查是否需要境外主体，及其注册成本和周期 |
> | 📅 目标市场法规变化 | 这12个月内目标市场有没有影响该商业模式的新法规、关税调整或平台政策变化？ | | 优先用SK 4.6（Claude裸跑）跑这项，它在实测中对政策变化最敏感 |

> - 政策变化查证 → 优先用SK 4.6（Claude裸跑），参见 `radar/signals.md` 末尾「深度研究AI工具组合」节
> - 供应链/平台成本 → DeepSeek
> - 官方费率/税率 → SK（GPTs深度研究员）

> 💡 **思维前置（每次扫描前默念）**
>
> **拆解一个产品的中国机会，不是问「这个产品能不能复制」，而是问「它提炼出的结构/模式，在哪些中国垂直里第一次成立」。
> 两个典型错误：
> - MYHAIR AI脉脱发诊断产品本身在中国可能竞争者已经很多→ 错误结论是「中国没有机会」
> - 正确结论是：「脱发诊断空白」这个**结构**，在皮肤/糖尿病前期/口腔这些垂直里同样存在且竞争者更少
> - Spangle产品本身在中国内贸平台电商不适用 → 错误结论是「中国没有P6机会」
> - 正确结论是：「意图续航」这个**结构**，在客服AI转坐席/直播→独立站等场景里是空白
>
> **演化路径**：拆解一个产品 → 提炼其底层结构/模式 → 扫描这个结构在哪些中国垂直里第一次出现 → 那里才是真正的机会窗口

> ⚠️ **第零步：基础设施前提检验（先做这个，再做下面所有分析）**
>
> 任何框架都有隐含的基础设施前提——那些框架成立所依赖的、往往被认为理所当然的环境条件。把框架迁移到中国市场之前，必须先问：
>
> **「这个产品/模式成立，依赖了哪些在原产地'理所当然'存在、但在中国未必存在的基础设施或用户行为路径？」**
>
> 检验步骤：
> 1. 写出原始产品成立的**行为路径**（用户怎么找到它、怎么使用它、为什么产生价值）
> 2. 逐段检查：这条路径的每一步，在中国的对应基础设施存在吗？
> 3. 如果有任何一步在中国不存在 → **直接写「结构性不适用，原因：___」，跳过下面所有分析**

>
> 典型案例（教训31）：Cactus依赖「Google本地搜索→消费者直接获得个体师傅电话→打过去」这条路径。这条路在美国是主干道，在中国城市市场被平台（高德/美团/大众点评）全面截断。结论：结构性不适用，不需要进入下面的中国机会分析。

> □ 通过——原始行为路径在中国存在，继续往下做
> □ **结构性不适用——原因**：___（填完直接跳到第10章四路决策）

## 9.5) 时间变量推敲（强制 · 四路决策之前）

**目的**：避免机会判断只看**当前静态条件**，忽略未来 **3–5 年**技术、成本、监管与用户习惯变化。**不是宏大预测**，只做轻量校准。

**位置**：已完成商业模式（第4章）与中国机会/时间窗口（第10-A章等）推演之后，进入 **第10章四路决策之前**，必须填写本节。

### 检查问题（内化后用 1–2 个变量落地）

1. **技术变量**：未来 3–5 年，是否有关键技术会变便宜、变强或变普及？（例：AI 推理成本、多模态、端侧 AI、机器人、检测设备家用化、生物技术突破）
2. **成本变量**：现在不成立是否主要因为太贵？若成本下降一个数量级，机会是被**打开**还是被竞争者**快速复制**？
3. **监管变量**：监管更可能打开、收紧，还是重塑玩家边界？
4. **用户习惯变量**：新工具或新疗法普及后，用户路径是否会改道？
5. **杀死变量**：是否存在单一变量会**显著压缩或杀死**当前叙事下的机会？

### 输出要求

- **只写 1–2 个最关键变量**；全文本节建议 **≤300 字**
- 不做长篇未来学；**不制造焦虑**
- 必须回答：该变量会 **打开 / 放大 / 压缩 / 改写 / 杀死** 这个机会？
- 若变量只**改写形态**，不得直接判定「机会消失」，须写出 **新位置 / 新切口**

### 模板（可直接粘贴改写）

### 时间变量推敲

这个机会不是静态的。

未来 3–5 年，最需要关注的变量是【变量A】和【变量B】。

如果【变量A】发生，它会【打开 / 放大 / 压缩 / 改写 / 杀死】这个机会，因为【原因】。

如果【变量B】发生，它会改变用户路径，使机会从【旧形态】改写为【新形态】。

所以当前判断不是「这个机会永远成立」，而是：
在当前阶段，【当前卡点】仍然存在；未来变量发生后，机会需要改写为【新位置】。

## 10) 四路决策——对我们意味着什么

## Step 1.5 「那一刀」确认（强制卡点，不可跳过）

> 填完上文各章后，必须停下来写出一句话：
>
> 「如果这篇文章只说一件事，让没见过这个产品的人10秒内决定『值得看』，那件事是：___」

- 写出来了 → 可进入对外输出（文章/PPT/讲稿等收口）
- 写不出来 → 回到材料收集补证据，或触发深度研究员；不进入对外输出

写完「那一刀」后，对照第 9 章理论映射，判断哪个理论框架能放大这个洞察（RRF / 财富公式 / 推演公式 / 破坏式创新 / 从0到1 等；RRF 边界同第 9b 章说明）。

### Step 1.5 辅助检查：致命变量（v2.0 新增）

写完「那一刀」之后，强制再问自己三个问题——如果答不上来，说明「那一刀」可能还停留在表面。

**问题 1**：这个产品所在行业，所有玩家都在优化的那个变量是什么？

> 参考例：咖啡 = 每杯履约成本；直播电商 = 退货率；新茶饮 = 单店模型；社区团购 = 生鲜损耗 × 履约成本

**问题 2**：这个产品/公司在这个变量上有没有突破？

> 如果「那一刀」说的不是它对致命变量的突破 → 你写的可能是表面特征
> 如果它在致命变量上没有突破 → 它的成功大概率是风口而非能力，不值得复制

**问题 3**：这个变量的天花板在哪里？还有多少优化空间？

> 天花板低 → 时间窗口快关了，别进场
> 天花板高 → 有机会，但要看是谁的机会（见第 10-A 章中国机会扫描）

**判断结果与「那一刀」的关系**：

- ✅ 「那一刀」正好说的是它对致命变量的突破 → 通过，进入 Step 2（对外输出）
- ⚠️ 「那一刀」说的是次级优势（如设计/体验/品牌） → 重写「那一刀」，把致命变量的突破放进去
- ❌ 该产品在致命变量上没有任何突破 → 可能是伪成功/风口产物，拆解时要重点做失败预警

- 致命变量的扫描方法详见 `radar/signals.md` 第 6 节「致命变量」
- 如果对致命变量判断不确定 → 触发深度研究员补数据（跨 3+ 家同行业公司对比）
