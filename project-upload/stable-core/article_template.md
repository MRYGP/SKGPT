## 开篇判断原则：30 秒规则

所有正式长文，默认应让读者在第一屏，也就是手机上约 300 字内，知道三件事：

1. 这篇要讲什么；
2. 作者的核心判断是什么；
3. 为什么值得继续读。

形式不限。

可以使用：

- `## 先说结论`
- `## 这篇的判断`
- `## 先把话说白`
- `## 我真正想说的是`
- 强引子开头；
- 第一段直接给判断；
- 故事开头但 300 字内落到判断。

核心不是有没有固定小标题，而是读者能否在 30 秒内说出：

> 这篇文章真正想说什么。

机会判断类文章，包括诊断空白、MTP、真实产品外部体检转招募、产品机会判断，建议显性前置判断卡，因为它们承担招募、验证和筛选同行者的任务。

但不要把“必须出现某个小标题”作为通用硬规则。

## 第二部分：结构参考（类型 A–H）

### 类型A：产品拆解（004-007, 012-014等）

> 拆一个具体AI产品。30篇里数量最多的类型。

```
标题：反直觉型 / 数字型 / 对比型（三选一）

开头（150字）：从读者已知的误解开始，不从产品介绍开始
  "所有人都以为[X]。但[产品名]的[N]个人做了[反常识的事]。"

一、产品是什么（200字）
  一句话定位 + 关键数据（A/B级）
  不写官网介绍语，写"我向朋友解释"的版本

二、反直觉洞察（400字）⭐ 给合伙人
  先说通常大家认为…… 再说这个产品的逻辑是……
  用数据或行为佐证

三、它怎么做到的（400字）
  拆1-2个关键机制
  有失败对比

四、时间窗口（300字）⭐ 给跟投人
  窗口还有多久 + 判断依据（具体信号）

五、小团队能学什么（300字）⭐ 给资源伙伴
  1-2个1-10人能执行的打法 + 不建议学的

结尾（100字）：三类人钩子 + 评论引导
```

**按需模块**（不是每篇都要，看"那一刀"需要什么）：

- 财富公式分析
- 推演公式分析
- RRF分析链
- 中国机会判断
- 四路决策

**原则：每次只装一颗子弹。**

#### 诊断空白与招募型叠加（底线，011–014 及相关 MTP/招募稿）

若本篇属于**诊断空白拆解**或**带招募意图**（含 MTP、外部体检转招募），须满足 **类型G · 机会招募型文章**中的「先说结论」五件事与「真位置前三分之一」硬规则；小节标题可用固定句式 **「这里真正空着的位置是什么」** 或并入 **「先说结论」** 内的「真位置」段。

该节须写明（可浓缩在同一段或一小节内，但不可整篇后置）：
- 真正缺的是哪一个**可做位置**（如第三方审查层、新角色、新交付单元），而不只是「行业不行」；
- 它与旧解法（现有诊所/平台/科普）差在哪；
- 潜在同行者为何应在当下回应作者（招募理由可对号入座）。

**位置硬约束**：**真位置**须在**用户卡点阐明之后**尽快落地，且落在全文**前三分之一**；不得只在后半篇才抛出「空位 / 卡位」。完整检查项见下 **类型G**。正文还须含 **`时间变量推敲`** 轻量段（见类型G第8步），与拆解侧 `product-teardown-template.md` §9.5、`SKILL-真实产品外部体检与机会推演SOP` Step 6.5 同源校准。

### 类型B：工具/公式教学（002, 003等）

> 教读者一个框架或方法。重点是"跑给你看"。

```
开头（150字）：一个问题或痛点
  "为什么[看起来成功的人/公司]其实[没有得到预期结果]？"

一、现象（300字）
  用1-2个案例展示问题
  正反对比（同类型，不同结果）

二、工具/公式（400字）
  介绍框架，用白话解释每一项
  不要超过3个要素（多了记不住）

三、跑给你看（500字）⭐ 核心段
  用一个真实案例当场演示
  最好用已发过的案例（读者有记忆锚点）

四、读者怎么用（300字）
  "如果你要判断[某个事情]，可以这样用这个工具"
  给一个具体的5分钟能做的动作

结尾：预告下一篇 + 评论引导
```

**教训3铁律：工具类文章没有"跑给你看"就是废文。**

### 类型C：失败案例/死法图谱（008等）

> 拆多个失败案例，提取共性死因。证明"我不只会夸好的"。

```
开头（150字）：一个触目惊心的数字
  "[N]个AI产品，每个融了千万美金，全死了。"

一、死法1（300字）
  [产品名]：做了什么→为什么死→FM编号
  一句话总结死因

二、死法2（300字）
  同上结构

三、死法3-5（根据篇幅，每个200-300字）

四、共性规律（400字）⭐
  "这5个死法，背后是同一个错误：[一句话]"
  回到推演公式/财富公式做解释

五、小团队怎么避免（300字）
  不是"小团队能学什么"，是"小团队怎么不死"
  具体的检查清单（3条以内）

结尾：评论引导
```

**注意：不是幸灾乐祸。语气是"我们从别人的代价里学到了什么"。**

### 类型D：身份/宣言/阶段总结（009, 016, 030等）

> "我是谁""我在做什么""我看到了什么"。不套拆解框架。

```
开头：一个具体的场景或时刻
  不从"我是XX"开始，从一个画面开始
  "那天下午，我盯着屏幕上的[某个东西]，突然意识到[某件事]"

正文：故事驱动，不是论证驱动
  允许暴露困惑和不确定
  理论自然带出，不列清单
  必须包含：我是谁、我在做什么、我在找谁

结尾：明确的行动邀请
  不是"欢迎关注"，是"如果你也在[具体方向]，来聊"
```

**教训4：这类文章放在读者已经通过拆解验证了你的判断力之后。009放在第9篇，不是第1篇。**

### 类型E：元分析/规律提炼（010, 022, 026等）

> 从多个已拆解案例中提炼共性规律。最难写的类型。

```
开头（150字）：一个数字或发现
  "拆完[N]个AI产品后，我发现了[N]条所有人都踩的坑"

一、规律1（400字）
  先说规律（一句话）
  再用2-3个已拆案例佐证（读者已经认识这些产品）
  对比：符合规律的成功了，违反的失败了

二、规律2（400字）
  同上结构

三、规律3（400字）
  同上

四、这意味着什么（300字）⭐
  "如果这三条规律成立，那[某个方向]的窗口比大家以为的[大/小]"
  连接到我们的押注

结尾：评论引导（让读者用这些规律去判断某个产品）
```

**关键：元分析的素材全部来自已发布的文章。每个规律至少引用2篇前文。读者感觉"他不是在凑数，背后有一个体系"。**

### 类F：MTP构思招募型（2026-04-19新增）

> **定义**：把一个还没做的产品构思，写成一篇能让B端资源人举手的公众号文章。
>  **和其他类型最大的区别**：其他类型在讲「已经发生的事」，F型在讲「我想做但还没做的事」，目的是提前验证资源是否存在。
>  **方法论文档**：`core/SKILL-MTP构思招募法.md`（必读，含7段骨架+10条检查清单）

```
标题：表达对象的具体状态，不是构思名称
  参考：「2.5亿中国人，每天照镜子，但没有一个人知道自己到底有多严重」

第1段：一个具体的卡住的人（300字内）
  有名字/年龄/具体生活场景，必须基于真实观察

第2段：所有人都在解错题（200字内）
  现有市场都在解什么题，真正卡住人的题没人解

第3段：这个构思要做的事（200字内）
  不写完整商业计划，只写核心动作一句话

第4段：这个构思能成立的前提（300字）
  3-5个关键假设+每个假设怎么验证——把推演暴露给读者

第5段：我可能踩的坑（2-3个自我攻击）
  这篇MTP和普通内容最大的区别——求被反驳

第6段：我在找什么样的人（极其具体）
  不是「欢迎交流」，是「如果你有X，私信告诉我Y」

第7段：如果没人举手（100字）
  明确承诺：没人举手就放弃+3个月后复盘
```

**核心与多数文章类型的区别**（如 A–E、G、H）：它们多在讲「已经发生的事」，F 型在讲「我尝试了还没做的事」。011 是三湘问道的第一个 MTP，虽然当时还没有这个名字。

### 类型G：机会招募型文章（2026-04-30 正式入库）

> **与开篇规则的关系**：招募稿同样必须先满足上文 **「开篇判断原则：30 秒规则」**；本节是在此之上，强建议显性前置判断卡，并强化 **真位置前三分之一** 与招募结构的叠加要求。

> **定义**：目标不仅是让读者承认「行业有问题」，更是让潜在同行者、行业内部人、产品人、资源人读完前半篇就能判断——**这里可能有一个新产品位置 / 新公司位置 / 新角色位置**，值得联系作者验证。
> **适用**：诊断空白系列；MTP 构思招募；真实产品外部体检（转招募）；阶段性押注宣言；任何以「寻找同行者 / 资源人 / 潜在合伙人」为显性目标的文章。
> **配套教训**：`content/公众号内容生产经验手册.md`（教训33–34）；第一读者「机会摘要检查」；写作工坊「机会招募型文章写作要求」。

**核心目的**：懂行读者读完**前三分之一**，能复述「这里可能做成什么」，而不止于「行业很乱」。

**标准结构（11 步，顺序可依题材微调，硬规则不可破）：**

1. **标题：问题 + 机会位置**  
   例：`同一个人的牙，8家诊所4种方案：口腔市场缺的不是诊所，是治疗方案的第三方审查层`

2. **开篇判断卡（建议显性）**  
   须在开头第一屏（约 300 字）优先回答下面五件事（可用一段话或分句，不可扔到篇末）：
   - 这篇**不是在**说什么（表面问题）；
   - **真正问题**是什么；
   - **真正空着的位置**是什么（可做 product slot / 新角色 / 新交付层）；
   - **为什么它可能商业化**（付费时刻、意愿、收费动作之一句钩子即可）；
   - **想找哪几类人**一起验证什么**最关键假设**。

3. **一个具体场景**：一个人 / 一案 / 一个冲突打开。

4. **真正卡住的不是表面问题**：重新定义问题（信任空白 vs 认知空白等）。

5. **这里空着一个位置**：浓墨打出**商业突破口**——不能只写痛点，必须写**可做位置**；须落在**前三分之一**（可与第2点合并呈现，但禁止整段后置到后半篇）。

6. **为什么现有玩家没占住**：利益错位、商业模型、监管、能力边界（择要写透）。

7. **为什么它可能是一门生意**：付费时刻、收费点、模式假设（区别于「免费导流」）。

8. **时间变量推敲（新增 · 看内容，不看字数）**  
   **位置**：紧接第7步之后，**风险与门槛（原第8步）之前**。  
   **写法**：只抓 **1–2 个**未来 **3–5 年**最关键变量（技术降本、监管、用户路径、疗法普及等）；说明会 **打开 / 压缩 / 改写 / 杀死** 中的哪一种；若仅改写形态，须写出 **旧位置 → 新位置**。短到一句话可以，复杂场景可更长。  
   **禁止**：宏大预测、长篇未来学、因变量可能存在而否定当前痛点、忽略明显的「杀死变量」。  
   **标准段落模板**：  
   > 这个机会不是静态的。未来 3–5 年，【变量A】和【变量B】可能会改变它。但它们不一定会消灭这个机会，更可能会改写它的形态。所以这篇文章讨论的不是「永远没人做的机会」，而是：在当前阶段，【当前卡点】仍然真实存在；如果未来变量发生，机会会从【旧位置】转向【新位置】。

9. **这件事难在哪里**：门槛写清，不画空饼。

10. **需要什么样的人**：招募对象可对号入座。

11. **下一步怎么验证**：MVP / 访谈 / 数据 / 评论区具体问题——结尾禁止只剩「欢迎交流」。

**硬规则**

- 禁止只写行业痛点，不写可做位置。
- 禁止把商业突破口放到后半篇才首次抛出。
- 禁止结尾只写泛泛「欢迎交流」，必须有**具体人群 + 具体验证命题 + 具体联系方式暗示**（私信我聊 X）。
- 懂行读者读完前三分之一，必须能复述：**这里可能做成什么**。

**固定开头模板（「先说结论」可直接粘贴改填空）**

```markdown
## 先说结论

这篇不是在说【表面问题】。

我真正想说的是：【真正问题】。

这里可能空着一个位置：【可做位置 / 新角色 / 新产品层】。

它可能成立，是因为：【用户已经到了某个高意愿时刻 / 现有玩家利益错位 / 旧解法解决不了】。

如果你是【具体三类人】，我想和你聊，因为这件事下一步需要验证：【最关键假设】。
```

### 可选模块：「如果你想试一下，这周的第一步」（2026-04-30 新增）

> **适用**：类 F（MTP）、类 G（机会招募）、类 H（真实产品外部体检）以及诊断空白类招募稿。不适用于纯拆解类（A）、工具教学类（B）、身份宣言类（D）、元分析类（E）。
>
> **为什么需要它**：招募类文章完成了「判断」那一半，但容易缺「召唤行动」那一半。读者读完能说「这个机会有意思」，但**不知道今晚可以做什么**。几天后热度冷下来，文章就只是「读过了」。「这周第一步」是动作钩子：给读者一个 0 成本、今晚能开始、做完后会对机会更中意 / 更不中意的小动作。

**位置**：文末「需要什么样的人 / 招募钩子」之前，独立一小节，默认标题 **「如果你想试一下，这周的第一步」**。

**内容三原则**

1. **0 成本**：不需要起项目、不需要拉人、不需要付费。用现成资源、现有权限就能开始。
2. **今晚能开始**：不是「未来三个月」，是「今晚 30 分钟」或「这个周末一个下午」。
3. **做完后能追加判断**：动作本身能让读者对机会的真实性拿到一手信号。可能是更中意，也可能是「原来不是这么回事」。两者都是有效产出。

**选人不选动作**

不要写「读者都可以做」的动作。要写「如果你是某类人，这周可以做」。不同人群可能三动作三起点。

**示例模板**

```markdown
## 如果你想试一下，这周的第一步

这篇文章讲的是一个机会，不是一个产品。但如果你读完觉得有点意思，下面是三种人 0 成本能动手的一件事。

**如果你是【行业内人】**：
这周可以做一件事——【动作，使用现有资源 / 数据 / 接触点】。做完后你会知道【某个判断是否成立】。如果你愿意，可以把结果发我。

**如果你是【潜在用户】**：
这周可以做一件事——【体验现有解决方案 / 记录一次自己的真实过程】。做完后你会知道【这个问题是否真的是问题 / 现有方案是否真的不够】。

**如果你是【可能的合作者 / 产品人】**：
这周可以做一件事——【拿这个问题去问 3-5 个真实用户 / 看一个同类产品怎么做】。做完后你会知道【这个方向是否值得下一步】。

做完哪一个都行。重点不是动作本身，是你在动作中会对这个机会有更真实的判断。
```

**检查单**

- [ ] 动作是否 0 成本？（需要起项目 / 拉人 / 付费 → 不合格）
- [ ] 动作是否今晚能开始？（需要准备超过 1 周 → 不合格）
- [ ] 动作完成后是否能让读者对机会拿到一手信号？（只是「多了解」不是信号）
- [ ] 是否针对不同人群给了不同动作？（一个动作跳过多样读者 → 低价值）
- [ ] 动作是否能被读者独立完成，不需要联系作者？（“联系我”不是动作，是招募钩子）

**这不是什么**

- 不是「招募钩子」的替代品：招募钩子仍需独立出现。
- 不是「完整方案」：不提供完整 12 个月路线图，只提供一个本周能动手的小事。
- 不是「教程」：SK 不是教程号。「这周第一步」不是「指导读者怎么做」，是「让读者自己验证机会」。

**反例（不要写这些动作）**

- 「加我微信聊聊」：这是招募钩子，不是动作。
- 「下载 XX 报告」：是消费不是验证。
- 「试用 XX 产品 30 天」：超过一周不算「这周第一步」。
- 「在评论区告诉我你的看法」：这是评论引导，不是动作。

**与“招募钩子”的区别**

这周第一步：**读者独立完成**，动作产出是「他自己对机会的判断」。
招募钩子：**读者与作者连接**，动作产出是「他与作者接上头」。

两个都要有。「这周第一步」在前，招募钩子在后。顺序不能反。

**待验证**

后续 5 篇招募类文章试用；如果连续 3 篇被读者实际动手验证过机会（评论区 / 私信 / 反馈出现「我试了」信号），考虑升级为招募类文章硬规则。

### 类型H：真实产品外部体检 + 机会推演 + 招募文

> **定义**：朋友/创业者发来真实早期产品，希望“从外部看后脑勺”。输出不是点评稿，而是体检报告+可迁移机会推演+是否公开点名建议。
> **方法论文档**：`core/SKILL-真实产品外部体检与机会推演SOP.md`
> **与类型G的关系**：若发文目标含招募，完稿时须叠加类型G的「先说结论」与真位置前置检查，不能只停留在体检报告体。

```
标题：先写真实问题，不先写产品名
  参考：「这不是产品不行，是叙事跑到了第三阶段」

第1段：先承认产品已做对什么（150-200字）
  讲优点和已成立的价值，避免“上来就判死刑”

第2段：外部误解在哪（200-300字）
  用户看不懂/看错的关键点是什么，误解如何产生

第3段：叙事错位诊断（300字）
  第一层价值 vs 长期愿景是否错位，证据来自官网/文档/定价/案例的一致性检查

第4段：可迁移机会推演（300-400字）
  哪些能力可迁移到更成立的场景，短中期分别怎么走

第5段：输出策略（200字）
  私下建议稿 / 脱敏公众号稿 / 点名公开稿，三选一并写判断依据

第6段：招募钩子（100-150字）
  如果要继续推演，需要什么角色的人来共创

第6.5段：时间变量推敲（内容审，不看固定字数）
  与未来3-5年技术/成本/监管/用户路径相关的1-2个关键变量；说明打开/压缩/改写/杀死；若转招募须与类型G第8步对齐
```

**关键边界**：外部体检不是“居高临下评价”，是“帮产品看见自己在外部的真实感知和可迁移路径”。

## 第四部分：选类型的决策树

```
这篇文章要：
├── 拆一个具体产品？ → 类型A
├── 教一个工具/公式？ → 类型B
├── 拆多个失败案例？ → 类型C
├── 说"我是谁/我在做什么"？ → 类型D
├── 从多个案例提炼规律？ → 类型E
├── 把一个还没做的构思对外招募B端资源？ → 类型F（必先读SKILL-MTP构思招募法.md）
├── 诊断空白 / MTP / 外部体检 / 押注宣言等，要强招募信号、前置商业突破口？ → 类型G（机会招募型：先说结论五件事 + 真位置前三分之一）
└── 对真实产品做外部体检并推演机会/判断是否点名？ → 类型H（必先读SKILL-真实产品外部体检与机会推演SOP.md；若转招募则叠加类型G）
```

如果一篇文章横跨两个类型（比如"用工具拆一个产品"），选主要目的那个。002的主要目的是教工具（B），不是拆papi酱（A）。
