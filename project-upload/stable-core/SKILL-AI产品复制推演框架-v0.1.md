# SKILL · AI产品复制推演框架
> 版本：v0.1
> 建立日期：2026-04-12，基于11x.ai推演过程提炼
> 迭代计划：每完成3个产品推演后更新一次版本
> 调用时机：收到一个海外AI产品，要判断"中国能不能复制/对标"时

---

## 使用说明

**什么时候用这个框架：**
- 收到一个海外AI产品，要判断中国能不能复制
- 在产品对标库里挑出来一个产品，要开始深度分析
- 有人带着"我想做类似XX产品的中国版"来讨论

**和其他工具的关系：**
- 本框架是**产品对标库**（`cases/2026/产品对标库-38个AI产品复制价值排名.md`）的配套执行工具
- 本框架跑完后，输出填入**10维度对标分析文件**（`cases/2026/深度底稿/产品名-对标分析.md`）
- 本框架是**推演SOP**（`core/SKILL-推演SOP-v1.3.md`）的前置步骤，推演SOP负责从方向到具体产品设计

**时间预估：**
- 有深度研究报告的情况下：1-2小时完成7步
- 没有报告需要临时搜索：3-4小时

## 框架全貌（7步）

```
Step 1：读懂"组合窗口"——它现在能做成，原来为什么做不了？
        ↓
Step 2：找最大死法——它踩了什么坑？是执行问题还是模式问题？
        ↓
Step 3：中国能不能直接复制？——按正确顺序问三个问题
        ↓
Step 4：找"已有关系"场景——中国市场哪个环节满足AI可接管的3个特征？
        ↓
Step 5：换题——重新定义问题，找到新的竞争对手
        ↓
Step 6：分渠道推演——针对中国主渠道逐一拆解产品形态
        ↓
Step 7：找差异化——对比现有CRM/工具，我们做的是什么它们做不了的
```

**每一步都有：问题、方法、输出格式、失败信号**

## Step 1：读懂"组合窗口"

### 核心问题
> 它现在能做成，原来（3-5年前）为什么做不了？
> 哪几件事同时成熟了？缺一件，这个产品就不成立。

### 方法
1. 列出产品依赖的所有技术/生态条件
2. 对每一个条件，问"2020年这个条件成熟了吗？"
3. 找出那个"最晚成熟"的条件，它就是打开窗口的关键变量
4. 验证：把任意一个条件去掉，产品还能成立吗？不能 = 找对了

### 输出格式
```
组合窗口描述：
- 条件A：[描述] → 成熟时间：[年份]
- 条件B：[描述] → 成熟时间：[年份]
- 条件C：[描述] → 成熟时间：[年份]
最晚成熟的条件：[C]，这是打开窗口的关键
一句话概括：[XX年，A+B+C同时成熟，[产品名]的窗口打开了]
```

### 失败信号
- 如果找不到"缺一不可"的条件组合 → 说明这个产品是渐进式改进，不是窗口型机会，降低优先级
- 如果所有条件都在5年前就成熟了 → 要问"为什么现在才做成功"，可能答案在执行力或市场教育，不在技术

### 11x.ai的例子
```
- 条件A：LLM个性化写作质量过阈值 → 2023年GPT-4发布后
- 条件B：B2B数据API生态成熟 → 2022年Apollo/ZoomInfo开放API
- 条件C：Agent编排框架出现 → 2023年LangChain等工具成熟
最晚成熟的条件：C，三件事缺一不可
一句话：2023年三件事同时成熟，AI SDR的窗口打开了
```

## Step 2：找最大死法

### 核心问题
> 它踩了什么坑？
> 这个坑是偶然失误（可以修正）还是模式本身的必然（绕不过去）？

### 方法
1. 找用户流失率和真实原因（不是官方说法）
2. 找退款/不续约的理由
3. 找竞争对手/行业观察者的批评
4. 找内部员工的匿名评价（Glassdoor等）
5. 判断：这是"做得不好"的问题，还是"这件事本质上就很难做好"的问题？

### 核心判断框架
```
凡是"重复、标准、信息处理"的动作 → AI做比人好，可以自动化
凡是"判断、关系、信任建立"的动作 → 人做比AI好，AI只能辅助

如果产品把AI放在了第二类动作上 → 死亡模式：预期落差型
如果产品把AI放在了第一类动作上 → 安全区
```

### 输出格式
```
最大死法：[描述]
死法类型：执行问题 / 定位问题（选一个）
证据：[来源]
是否可以通过重新定位规避：是/否
如果是，规避方式：[描述]
```

### 失败信号
- 如果找不到任何批评声音或失败案例 → 产品可能太新，数据不足，需要标注"待验证"
- 如果死法完全是创始人个人问题（欺诈/管理混乱）→ 不算模式失败，这个产品方向可能还有机会

### 11x.ai的例子
```
最大死法：流失率70-80%，AI约会质量不达预期，客户3个月不续约
死法类型：定位问题
证据：TechCrunch 2025年3月调查报道，前员工匿名披露
是否可规避：是
规避方式：从"AI替代SDR"改为"给SDR的AI杠杆"，降低预期，变成Copilot模式
```

## Step 3：中国能不能直接复制？

### 核心问题
> 不要先问"有没有机会"，要先问"用户为什么不直接用原版"

### 必须按顺序问的三个问题

**问题一：中国用户为什么不直接用原版？**
检查清单：
- [ ] 价格壁垒？（原版太贵，中国用户付不起）
- [ ] 语言壁垒？（没有中文版，使用门槛高）
- [ ] 渠道壁垒？（原版依赖的渠道在中国不存在，如LinkedIn/Gmail）
- [ ] 合规壁垒？（数据跨境/支付/牌照问题）
- [ ] 文化壁垒？（业务场景在中国不存在或逻辑不同）

**如果以上都没有 → 直接复制没有护城河，停止，返回重新定义问题**

**问题二：如果有壁垒，我们能真正解决它吗？**
- 语言壁垒 → 做本地化，技术门槛低，竞争会很快出现
- 渠道壁垒 → 这里往往藏着真正的机会（见Step 4）
- 合规壁垒 → 需要资质/资源，小团队谨慎

**问题三：中国的使用渠道和原版有什么本质不同？**
这个问题往往是发现真正机会的入口。

### 输出格式
```
直接复制可行性：可行 / 不可行 / 部分可行
主要壁垒：[列出]
渠道差异地图：
  原版依赖的渠道：[如邮件/LinkedIn]
  中国对应的渠道：[如微信/电话]
  渠道差异带来的机会：[描述]
结论：
  直接复制 → 进入Step 6
  需要重新定义 → 进入Step 4/5
```

### 失败信号
- 如果渠道差异很大，但你没有想到如何利用 → 不要跳过，这里很可能是真正的机会所在
- 如果壁垒只是"价格"，且差距不大 → 护城河极弱，慎重

### 11x.ai的例子
```
直接复制可行性：不可行（外贸场景）
主要壁垒：外贸公司可以直接用国外工具，没有明显壁垒
渠道差异地图：
  原版依赖：英文邮件/LinkedIn
  中国主渠道：企业微信/电话营销
  渠道差异带来的机会：国外工具完全不能覆盖微信/电话场景
结论：需要重新定义问题，进入Step 4
```

## Step 4：找"已有关系"的跟进场景

### 核心洞察（从11x推演中提炼）
> 陌生触达：合规风险高 + 转化率低 + 竞争激烈
> 已有关系的跟进：合规风险低 + 转化率高 + 竞争相对少

大多数AI产品在"陌生触达"环节竞争，但"已有关系的跟进"是一个更大、更安全、更少人做的市场。

### AI可以接管的判断标准（满足3个特征）
1. **已有客户关系**：不是陌生触达，有过互动
2. **内容标准化**：核心内容固定，只是姓名/金额/时间等变量不同
3. **有明确的触发条件**：时间到了/事件发生了/行为触发了

**银行AI电话就是完美案例：已有关系+标准化内容+事件触发（账单日）**

### 方法
在目标行业的销售/服务流程里，找出满足以上3个特征的所有环节，列出来。

### 输出格式
```
可能的AI接管场景列表：
场景1：[描述] | 已有关系✅/❌ | 标准化✅/❌ | 触发条件✅/❌ | 综合判断
场景2：...
场景3：...

优先场景：[最满足3个特征的那个]
原因：[为什么这个最好]
```

### 失败信号
- 如果这个行业所有核心动作都是"非标准化的判断动作" → 这个行业可能不适合AI跟进，考虑换行业
- 如果已有大量成熟产品在做这件事 → 重新评估差异化空间

### 11x.ai的例子
```
场景1：已有线索的跟进确认电话 | ✅ | ✅ | ✅（3天未回复触发）| 最优先
场景2：朋友圈定期发内容 | ✅ | 部分✅ | ✅（定期触发）| 可以做
场景3：报价后催单 | ✅ | ✅ | ✅（7天未回触发）| 可以做
优先场景：已有线索的跟进确认电话（场景B类型）
```

## Step 5：换题（重新定义问题）

### 核心问题
> 他们在解决什么问题？
> 中国市场最相似但又最不同的问题是什么？

### 换题成功的信号
**当"竞争对手"从"其他AI公司"变成了"用户的某个习惯或限制"，换题成功。**

- 换题前：竞争对手 = 全球同类AI产品
- 换题后：竞争对手 = 用户的记忆力/时间/某个低效习惯

### 方法：问三层"为什么"
```
原产品解决的问题：[A]
→ 为什么用户有问题A？因为[B]
→ 为什么中国用户的B和美国用户的B不一样？因为[C]
→ 如果我们直接解决C，而不是A，竞争对手是谁？
```

### 输出格式
```
原问题：[他们在解决什么]
新问题：[我们重新定义后的问题]
换题理由：[为什么这个新问题更适合中国市场]
新的竞争对手：[不再是AI公司，而是什么]
验证：如果我们解决新问题，原来的竞争对手还是我们的对手吗？是/否
```

### 失败信号
- 如果换了题之后，竞争对手反而更多、更强 → 题换错了，回到原问题或者换个方向
- 如果新问题太宽泛（"提升效率"类） → 继续缩小，直到能说出一个具体的竞争对手

### 11x.ai的例子
```
原问题：怎么自动化触达更多陌生潜在客户
新问题：怎么让一个销售管好他手里现有的200个线索
换题理由：中国销售的核心痛点不是找不到客户，是找到了跟不过来
新的竞争对手：销售的记忆力和时间（不再是AI SDR公司）
验证：Artisan/11x等AI SDR不再是竞争对手 ✅
```

## Step 6：分渠道推演产品形态

### 核心问题
> 针对中国每个主要触达渠道，AI能做什么、不能做什么？
> 最小可行的产品形态是什么？

### 渠道评估四问（对每个渠道都问一遍）
1. **API开放程度**：这个渠道的自动化上限在哪里？
2. **可AI接管的动作**：哪些是"重复+标准+触发"的？
3. **必须人工的动作**：哪些需要判断和信任？
4. **MVP形态**：最小可行产品是什么？

### 中国主要渠道速查表（通用参考）

| 渠道 | API开放度 | 自动化上限 | 主要限制 |
|------|---------|----------|--------|
| 企业微信 | 半开放 | 可读取记录/发送模板消息 | 不能自动发个人消息，需人工确认 |
| 个人微信 | 基本封闭 | 极低，封号风险高 | 不建议自动化 |
| 电话（外呼） | 取决于场景 | 已有关系+标准化场景可合规 | 陌生外呼监管严格 |
| 短信 | 开放（企业） | 通知类可以 | 营销短信有限制 |
| 邮件 | 完全开放 | 高 | 国内打开率低 |
| 钉钉 | 企业内部开放 | 较高（企业内部场景）| 限企业内部 |

### 输出格式（对每个渠道）
```
渠道：[名称]
API限制：[具体限制]
AI可接管的动作：[列出]
必须人工的动作：[列出]
合规风险：高/中/低
MVP形态：[一句话描述最小产品]
启动成本：[预估]
```

### 失败信号
- 如果所有渠道的API都封闭 → 这个方向在中国技术上很难做，重新评估
- 如果MVP需要依赖某个单一渠道的特定API → 标注平台依赖风险（FM-006）

### 11x.ai的例子
```
渠道：企业微信
AI可接管：读聊天记录→判断状态→生成草稿→提醒销售
必须人工：发出消息（合规关键）
MVP形态：AI分析聊天记录，生成"今天该联系谁+说什么"的优先级列表

渠道：电话（跟进确认型）
AI可接管：场景B——已有接触的客户3天未回复后的跟进确认电话
必须人工：复杂问题转人工
MVP形态：事件触发→AI自动打跟进确认电话→录音转文字→更新客户状态
```

## Step 7：找差异化

### 核心问题
> CRM/现有工具已经在做"提醒"了，我们和它们的区别是什么？

### 差异化的本质判断
```
现有工具做的是：提醒你去做某件事
我们做的是：帮你想好做什么，并且帮你做出去

这中间的执行落差 = 差异化空间
```

### 方法：找"最后一公里"
1. 现有工具做到哪里停了？
2. 用户从工具提醒到真正执行，中间还要做哪些事？
3. 这中间哪一步最耗时、最容易被跳过？

### 输出格式
```
主要现有竞品：[列出2-3个]
他们做到哪里停了：[描述]
用户还需要自己做的事：[列出]
我们能帮用户做的：[列出]
一句话差异化：[不超过20字]
他们为什么不做这件事：[原因——技术/商业模式/定位]
```

### 失败信号
- 如果差异化只是"做得更好" → 不够，竞品很快能追上
- 如果差异化依赖某个对手不会有的独特资源 → 这是真正的壁垒，深入挖掘
- 如果说不清楚"他们为什么不做" → 差异化可能不成立

### 11x.ai的例子
```
主要竞品：销售易、纷享销客、飞书CRM
他们做到哪里停了：提醒你"该联系XX客户了"
用户还需要自己做的：想说什么、怎么说、然后手动去说
我们能帮用户做的：分析聊天记录→判断客户状态→生成个性化消息草稿→
                  （电话场景）直接帮你说出去
一句话差异化："他们提醒你跟进，我们帮你跟进"
他们为什么不做：CRM公司的核心是数据管理，不是AI生成；
              做AI生成需要LLM调用能力+对话理解，超出了他们的产品定位
```

## 框架完成后的输出

跑完7步，应该能回答以下问题：

```
1. 这个产品的"组合窗口"是什么？中国版的窗口还开着吗？
2. 原版的最大死法是什么？我们能规避吗？
3. 中国版的产品形态应该是什么样的？（不是原版的翻译，是重新定义后的版本）
4. 目标用户是谁？（具体到职业+场景+痛苦）
5. 一句话差异化是什么？
6. 四路决策：复制/工具/代理/观察？
```

把这些答案填入 `产品对标库` 的10维度表格。

## 触发更新的条件

以下任一情况触发版本更新：
- 完成3个新产品的推演后
- 某一步的失败信号与预期完全不符时
- 发现某个步骤在新产品上完全走不通时
- 用户访谈带来了和桌面推演相反的结论时
